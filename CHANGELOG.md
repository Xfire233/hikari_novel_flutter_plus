# Changelog

## 1.2.1+53

- 修复 Wenku8 兼容模式下 `reader.php?aid=...` 目录入口被跳转到静态目录网关时，WebView 只拿到短网关页并最终超时的问题。
- 增加目录入口网关页的静态目录链接跟随逻辑，同时保留目录/章节 HTML 校验，避免错误页或详情页被误当作目录缓存。

## 1.2.0+52

- Wenku8 兼容模式改为自动 WebView 抓取链路，不再在页面错误态弹出单独浏览器验证入口。
- Wenku8 Cloudflare / 兼容模式失败页只保留“重试”和“开启兼容性模式”，并保留 URL 与 WebView 状态，便于截图反馈。
- 修复 Wenku8 目录抓取可能把详情页 HTML 提前当作目录页返回的问题，降低 `Wenku8 catalogue is empty` 误报。
- 兼容模式加载中增加等待提示，明确告知用户当前正在走 WebView 抓取。

## 1.1.6+35

- 首页下拉刷新改为操作逻辑锁：顶栏收起、展开中、或当前下拉手势尚未结束时，刷新组件本身禁用刷新入口，避免刷新 header 与顶栏展开同时触发。
- Wenku8 / ESJZone / Yamibo 首页统一使用同一套刷新锁，顶栏完全展开且稳定后，下一次下拉才允许触发网络刷新。

## 1.1.6+34

- 放慢首页分区栏和筛选栏收起动画，筛选栏收起时保留内容渐隐，避免瞬间消失造成误判。
- 首页下拉刷新增加顶栏展开门槛：顶栏未展开或刚展开时只展开导航，顶栏稳定展开后再次下拉才触发网络刷新，降低 Wenku8 等首页误刷新。
- 修复切换来源后第一次点击分区导航偶发无响应的问题，收口 PageController pending 同步逻辑。
- 阅读器在翻页、切章、退出和应用进入后台时立即写入阅读进度，并同步刷新最近阅读时间；Wenku8 加入书架时先写本地书架，避免远端同步过程中后台退出导致本地状态丢失。

## 1.1.6+33

- 三来源首页分区统一支持点击和左右滑动切换，AppBar 高亮会跟随手势和点击动画平滑过渡。
- Wenku8、ESJZone、Yamibo 首页右上角菜单统一为来源状态、登录检查和在线收藏同步；点击状态栏打开对应来源账号网页，首页刷新统一交给下拉刷新。
- Wenku8 首页菜单增加 Cloudflare 辅助验证入口，ESJZone / Yamibo 不再保留重复手动刷新按钮。
- Wenku8 登录改为用户确认后返回首页，由首页统一刷新内容和触发书架同步，避免登录后灰屏且必须重启应用。
- ESJZone / Yamibo 登录确认后同样返回对应来源首页并按设置同步来源书架；关闭 WebView 只同步 cookie，不误触发书架同步。
- 书架根目录中的 Wenku8 来源书架显示名恢复为 `Wenku8`，不再显示为默认书架。
- 首页来源切换改为独立 PageController，ESJ/Yamibo 分区懒加载，减少切换红屏和并发请求风险。
- 我的页登录入口复用首页登录后处理逻辑，补齐刷新反馈、核心按钮语义和控制器释放。

## 1.1.5+31

- Android 包名和应用显示名统一改为 Hikari Novel Plus，与当前仓库命名保持一致。
- Yamibo 智能订阅改为深度匹配：候选会检查标题、论坛元信息、首楼简介和楼主正文，命中的用户标签会写入本地书籍信息并在详情刷新后保留。
- 智能书架新增替换 / 增量同步模式；每个书架支持单独同步，并在同步时显示封面或列表遮罩进度。
- 修复 Yamibo / ESJ 登录状态判断：同步前做真实登录校验，登录失效时提示用户，WebView 关闭不再误触发同步或清空智能书架内容。
- 增加 Yamibo 每日 05:30-06:00 论坛备份窗口检测；备份期间无数据返回会显示明确提示。
- Wenku8 增加 Cloudflare / 浏览器辅助 fallback，并修复错误页导致的红屏崩溃。
- 统一来源首页、书架、子书架、列表和下拉菜单样式，改为圆角胶囊 / 卡片布局，并补齐主要页面切换动画。
- About 页 GitHub 链接更新为 `https://github.com/Xfire233/hikari_novel_flutter_plus`，移除 Telegram 入口。
- 增加本地 Android 模拟器与 uiautomator2 MCP 调试流程，减少后续重复配置和人工传包验证。
