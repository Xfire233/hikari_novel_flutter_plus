# Hikari Novel Plus 叉烧魔改版

> 基于 [15dd/hikari_novel_flutter](https://github.com/15dd/hikari_novel_flutter) 的独立 fork。当前测试版本：`beta1.1.5-pre.2`。

这是一个围绕 Hikari Novel 大幅改造的多来源阅读器。原项目主要服务 Wenku8，本 fork 追加 ESJZone 和 Yamibo，并重做了来源启用、登录、书架、搜索、收藏同步、阅读器和电子墨水屏浏览逻辑。当前项目不以向原项目提交 PR 为目标，只作为独立 fork 使用。

## 重要声明

- 本项目仅供个人学习、交流和测试使用。
- 大量功能、代码整理、文档和 UI 改造由 AI 辅助生成或直接生成，人工审阅和实机覆盖不足。
- 当前仍存在缺陷、兼容性问题和架构债，不应视为稳定客户端。
- 本项目与 Wenku8、ESJZone、Yamibo 及原 Hikari Novel 项目均无官方关系。
- 请尊重来源网站规则，不要高频抓取、批量下载或公开分发编译产物。

## 相较原项目的关键变化

- 新增 ESJZone 来源：浏览、详情、章节阅读、收藏同步、阅读进度导入。
- 新增 Yamibo 来源：文学区原生列表浏览、子区切换、关键词搜索、作者主题列表、收藏拉取。
- 解除 Wenku8 强制登录启动逻辑：三个来源可以独立启用、登录、同步和配置。
- 统一本地书架：来源默认书架、子书架、最近阅读、智能书架、订阅标签书架、排序记忆和本地评分。
- 在线收藏同步：默认从网页端收藏拉取到本地；本地回写需要单独开启，尽量避免干扰网页端收藏夹。
- 阅读器适配：ESJZone / Yamibo 接入本地阅读器、阅读进度、章节缓存、图片查看、点按翻页和音量键翻页。
- 电子墨水屏适配：阅读页和来源浏览页支持分页、音量键翻页，并减少滚动动画。
- 搜索重构：只显示已启用来源，各来源保留独立搜索和高级筛选。
- 本地备份：JSON 单文件备份 Cookie、应用设置、书架、阅读记录和评分。

## 当前功能状态

- Wenku8：保留原有核心阅读链路，接入新书架和多来源设置。
- ESJZone：可用于浏览、阅读和收藏同步；网页结构变化可能导致解析失败。
- Yamibo：文学区浏览、搜索、收藏拉取和楼主楼层阅读可用；收藏回写仍谨慎处理。
- 书架：支持列表 / 网格、单独排序、子书架、智能条件、订阅标签、封面 fallback 和本地评分。
- 阅读器：支持滚动、翻页、双页、点按翻页、音量键翻页、电子墨水屏模式和全面屏避让。
- 主题：支持应用主题、阅读主题、来源图标和来源背景水印。

## 已知问题

- AI 生成代码较多，局部逻辑可能存在未发现的运行时问题。
- 多来源收藏回写仍有风险，使用前请先备份。
- iOS 未测试，不保证可编译、可登录或可正常保存 Cookie。
- Windows 端不是主要目标，体验不完整。
- ESJZone / Yamibo 页面结构变化会直接影响解析。
- 电子墨水屏适配只在少量 Android 设备上验证过。

## 使用说明

1. 首次启动选择需要启用的来源，之后可在“设置 > 来源设置”中调整。
2. Wenku8 通常需要登录后才能正常访问。
3. ESJZone 和 Yamibo 需要先通过应用内网页登录，再同步收藏。
4. 默认同步策略是“在线收藏 -> 本地书架”，不会修改网页端收藏夹。
5. 如需把本地变动回写到网页端，请在对应来源设置中手动开启回写策略。
6. 备份文件可能包含 Cookie，请勿公开分享。
7. 同包名覆盖安装不会清理本地数据库、Hive 设置、书架、阅读记录和缓存；卸载应用或清除应用数据会丢失本地数据。

## 构建

推荐环境：

- Flutter SDK 3.38.x
- Android Studio JBR 17
- Android SDK 36.x

Android release APK 构建示例：

```powershell
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'
$env:Path="C:\Windows\System32\WindowsPowerShell\v1.0;C:\flutter\bin;$env:JAVA_HOME\bin;$env:Path"
C:\flutter\bin\flutter.bat build apk --release --no-pub
```

发布测试包命名约定：

```text
Hikari-Novel-Modified-版本号.apk
```
